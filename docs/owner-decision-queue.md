**Spec gap ที่ต้องตัดสินเชิง design (สำคัญกว่าตัวเลข):**

| # | เรื่อง | ปัญหา | ตัวเลือก |
|---|---|---|---|

## หมวด 4: Art & Asset Pipeline (บันทึกจากที่คุย 2026-07-12 + คำถามย่อย)

**เคาะแล้ว (บันทึกไว้):** เฟสนี้ไม่มีทีมกราฟฟิค → tech ทำ **placeholder สไตล์ pixel-art-like** เองไปก่อน (วาดบน grid เล็ก ขอบคม จำกัดสี ขยาย nearest-neighbor) เอาความไวเป็นหลัก · เมื่อได้ pixel art จริงค่อยทยอย replace — ระบบ animation manifest เป็น data-driven อยู่แล้ว สลับภาพได้โดยไม่แตะ logic

**คำถามย่อยที่เหลือ:**

| # | เรื่อง | ต้องเคาะ |
|---|---|---|
| 4.1 | **ลำดับความสำคัญ asset** | เสนอ: ตัวละคร 5 อาชีพ → มอน Map 1 → เมือง/props → เอฟเฟกต์สกิล → บอส — เรียงตามนี้ไหม |
| 4.2 | **มาตรฐานขนาด** | ตัวละครกี่ px (เสนอ ~32×48 บน tile 64×32) · มอนไซซ์เล็ก/กลาง/ใหญ่ — ล็อกครั้งเดียวใช้ตลอด กัน art จริงมาแล้วสเกลเพี้ยน |
| 4.3 | **จานสีกลาง (palette)** | ให้ tech กำหนด palette จาก ref กลุ่ม A แล้วใช้ทั้งเกมเลยไหม (ทำให้ placeholder ทุกชิ้นโทนเดียวกัน + art จริงมี target ชัด) |
| 4.4 | **Format ส่งมอบ art จริง (บอกทีมกราฟฟิคล่วงหน้าได้)** | sprite sheet PNG + ผังเฟรมตาม animation manifest ของเรา (เดิน/idle/ตี × 5 ทิศ+mirror) — tech จะร่าง spec sheet ให้เมื่อเคาะ 4.2 |
